# Repo Review Plan

Branch: workbook-os/scaffold-v1

## Review method

1. Review this branch before merge.
2. Keep main unchanged unless Jeremy approves merge.
3. If scaffold review fails, leave the branch unmerged and record the reason in receipts.
4. Any cleanup should happen on the scaffold branch first.

This scaffold adds root rules, manifest, folder placeholders, workflow stubs, script stubs, and templates. It does not deploy production or change live services.
