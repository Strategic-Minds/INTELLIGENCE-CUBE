# Visual Parity Addendum

Visual parity means the final website/app/dashboard matches the approved visual source truth.

Required evidence:
- Approved mockup / design source
- Desktop screenshot
- Tablet screenshot
- Mobile screenshot
- Visual drift ledger
- UI parity scorecard
- Operator visual approval or exception receipt

Release rule:
No UI project may release with unresolved P0/P1 visual drift.
