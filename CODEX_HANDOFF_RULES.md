# Codex Handoff Rules

Codex operates from the workbook, not from loose assumptions.

## Required reading

1. README.md
2. WORKBOOK_ONLY_RULES.md
3. WORKBOOK_OS_MANIFEST.json
4. GPT_START_HERE.md
5. latest workbook index
6. master workbook
7. relevant child workbook
8. receipts and blockers

## Operating limits

- Work on branch workbook-os/scaffold-v1 unless another approved branch is named.
- Do not push directly to main.
- Do not deploy production.
- Do not remove existing files unless an approved workbook or issue explicitly says so.
- Update receipts and rollback notes for every change.
- If a required workbook is missing, create a blocker instead of inventing requirements.

## Purpose

Create or update only the scaffold files, folders, scripts, workflows, and workbook-referenced support artifacts specified by the approved workbook or issue.
